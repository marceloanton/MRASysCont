# Security Policy

## Supported Versions

Se da soporte de seguridad sobre `main` y la ultima version desplegada.

## Reporting A Vulnerability

No abrir vulnerabilidades en issues publicos.

Reportar por email a:
- marceloanton@outlook.com

Incluir:
- descripcion del hallazgo,
- pasos para reproducir,
- impacto,
- alcance (study/company/data),
- mitigacion sugerida.

## Response Targets

- Acuse de recibo: dentro de 72 horas.
- Triage inicial: dentro de 7 dias.
- Plan de remediacion: segun severidad.

## Scope Prioritario

- Fugas multi-tenant.
- BOLA (autorizacion por objeto rota).
- Exposicion documental sin permisos.
- Secretos/token/certificados en logs.
- Bypass de reglas contables criticas.
